# CLAUDE.md

## 목적
이 문서는 작업 수행 시 반드시 따라야 하는 기본 작업 규칙을 정의한다.  
모든 작업은 아래 절차를 기준으로 수행한다.

---

## 작업 원칙

### 1. 작업 시작 전 계획 수립
- 실제 작업에 들어가기 전에 반드시 `TODO.md`에 작업 계획을 작성한다.
- 계획 없이 바로 구현하거나 수정하지 않는다.
- 계획에는 작업 목표, 세부 작업 항목, 예상 영향 범위를 포함한다.

### 2. 작업 계획서 작성 후 진행
- 작업 시작 전 반드시 작업 계획서를 작성한다.
- 작업 계획서는 `TODO.md`에 작성한다.
- 작업자는 작성된 계획서를 기준으로만 작업을 진행한다.
- 작업 중 범위가 변경되면 계획서를 먼저 갱신한 뒤 작업을 이어간다.

### 3. 계획서 기반 작업 수행
- 모든 구현, 수정, 리팩터링, 문서화 작업은 사전에 작성한 계획서를 근거로 진행한다.
- 계획에 없는 작업을 임의로 추가하지 않는다.
- 예외적으로 추가 작업이 필요하면 `TODO.md`에 먼저 반영한 후 진행한다.

### 4. 작업 완료 후 보고서 작성
- 작업이 끝나면 반드시 작업 완료 보고서를 작성한다.
- 보고서에는 다음 내용을 포함한다.
    - 작업 개요
    - 실제 수행한 내용
    - 변경된 파일 또는 주요 영향 범위
    - 이슈 및 특이사항
    - 후속 작업 필요 여부

### 5. TODO.md 최종 정리
- 작업 완료 후 root 경로의 `TODO.md`에 작업한 내용을 간결하게 정리한다.
- 최종 정리는 불필요하게 길지 않게 작성한다.
- 누구나 빠르게 확인할 수 있도록 핵심만 남긴다.

---

## 작업 절차

1. root 경로의 `TODO.md`에 작업 계획 작성
2. 계획 검토
3. 계획서 기준으로 작업 수행
4. 작업 완료 후 보고서 작성
5. root 경로 `TODO.md`에 작업 결과 간결하게 정리

---

## TODO.md 작성 규칙

### 작업 전
아래 형식으로 계획을 작성한다.

```md
# TODO

## 작업 계획
- 작업 목표:
- 세부 작업:
  - 
  - 
  - 
- 영향 범위:
- 검토 사항:

## 진행 메모
- 
```

### 작업 후
아래 형식으로 결과를 정리한다.

```md
## 작업 보고서
- 작업 개요:
- 수행 내용:
- 변경 파일:
- 이슈/특이사항:
- 후속 작업:

## 최종 정리
- 작업 내용 요약:
- 완료 여부:
```

---

## 금지 사항
- `TODO.md` 작성 없이 바로 작업 시작 금지
- 계획서 없이 임의 수정 금지
- 작업 완료 후 보고서 누락 금지
- 작업 결과를 root 경로 `TODO.md`에 정리하지 않고 종료 금지

---

## 기본 원칙 요약
- 먼저 계획 작성
- 계획에 따라 작업
- 끝나면 보고서 작성
- 마지막으로 `TODO.md`에 간결하게 정리
